# 从单一 MCP 迁移到多 MCP 架构指南

## 📋 迁移概述

本指南帮助你从原始的 `oilfield_mcp_true_server.py` 平滑迁移到新的多 MCP 架构。

---

## 🔄 架构变化

### **旧架构** (单一 MCP)
```
oilfield_mcp_true_server.py (1801行)
├── 8个工具
├── 端口: 8081
└── 所有功能在一个文件中
```

### **新架构** (多 MCP)
```
common/ (共享模块)
├── db.py
├── permissions.py
├── utils.py
└── audit.py

oilfield_wells_mcp.py (端口 8081)
└── 5个工具：油井基础数据

oilfield_dailyreports_mcp.py (端口 8082)
└── 3个工具：日报系统
```

---

## 🔧 迁移步骤

### 步骤 1：停止旧服务
```powershell
# 如果旧服务正在运行，先停止它
Get-Process | Where-Object {$_.MainWindowTitle -like "*oilfield*"} | Stop-Process
```

### 步骤 2：备份旧配置（已完成）
旧文件已保留为 `oilfield_mcp_true_server.py`，不会被删除。

### 步骤 3：测试新服务
```powershell
# 运行测试脚本
.\test_mcps.ps1
```

### 步骤 4：启动新服务
```powershell
# 启动所有新 MCP 服务器
.\start_all_mcps.ps1
```

### 步骤 5：更新 LibreChat 配置

**旧配置（删除或注释）:**
```json
{
  "mcpServers": {
    "oilfield-true": {
      "command": "python",
      "args": ["d:\\work\\oilMCP\\oilfield_mcp_true_server.py"],
      "env": { ... }
    }
  }
}
```

**新配置（添加）:**
```json
{
  "mcpServers": {
    "oilfield-wells": {
      "command": "python",
      "args": ["d:\\work\\oilMCP\\oilfield_wells_mcp.py"],
      "env": {
        "DB_HOST": "localhost",
        "DB_PORT": "5432",
        "DB_NAME": "rag",
        "DB_USER": "postgres",
        "DB_PASSWORD": "postgres",
        "DEV_MODE": "true"
      }
    },
    "oilfield-dailyreports": {
      "command": "python",
      "args": ["d:\\work\\oilMCP\\oilfield_dailyreports_mcp.py"],
      "env": {
        "DB_HOST": "localhost",
        "DB_PORT": "5432",
        "DB_NAME": "rag",
        "DB_USER": "postgres",
        "DB_PASSWORD": "postgres",
        "DEV_MODE": "true"
      }
    }
  }
}
```

### 步骤 6：重启 LibreChat
重启 LibreChat 使新配置生效。

## 🐛 常见问题

### ❓ 旧服务还能用吗？
**答**: 可以，但不推荐。旧服务文件已保留，但建议使用新架构。

### ❓ 如何回滚到旧架构？
**答**: 
1. 停止新服务
2. 恢复旧的 LibreChat 配置
3. 启动旧服务：`python oilfield_mcp_true_server.py`

### ❓ 数据会丢失吗？
**答**: 不会。新旧架构使用同一个数据库，数据完全一致。

### ❓ 性能有提升吗？
**答**: 是的。新架构启动更快，响应更快，资源占用更低。

### ❓ 如何同时运行新旧服务？
**答**: 
```powershell
# 旧服务使用 8081
python oilfield_mcp_true_server.py

# 新服务使用 8082, 8083（需修改端口配置）
# 不推荐这样做，容易混淆
```

## 🎯 最佳实践

### 1. 渐进式迁移（推荐）
- 先在测试环境验证新架构
- 确认无问题后再迁移生产环境
- 保留旧配置作为备份

### 2. 并行运行（谨慎）
- 可以短期内同时运行新旧服务
- 但容易造成端口冲突
- 不建议长期使用

### 3. 一次性迁移（快速）
- 直接停止旧服务
- 启动新服务
- 适合对系统熟悉的用户
